3) 공통 UI 스타일 가이드
3.1 톤&무드

Typography: 산세리프(예: Inter/pretendard 계열)

카드: 둥근 라운드(12~16px), 미세한 그림자/보더

버튼: 동일한 높이/폭, 아이콘 optional(기본은 아이콘 포함 추천)

3.2 반응형

Mobile first

Breakpoint 예시:

Mobile: 360~767px (카드 폭 92~94%)

Tablet/PC: 카드 max-width 420~520px, 중앙 정렬

3.3 접근성

버튼/토글은 키보드 포커스 링 표시

대비(다크/라이트) 확보

4) 기능 요구사항 (FR)
FR-1: 링크 버튼(4종)

버튼 4개 고정 노출(기본값)

클릭 시 동작: 새 탭 열기(target="_blank", rel="noopener noreferrer")

링크 URL은 설정값(상수/ENV/간단한 JSON config)로 주입

링크 목록

오픈 카카오톡 링크 (Open Chat URL)

유튜브 링크 (채널/영상/플레이리스트 가능)

스레드 링크 (threads.net/...)

링크드인 링크 (profile/company 가능)

링크 누락 처리(기본값 선택)

“don’t care”이므로 표준 기본안:

URL이 비어있으면 버튼 숨김(가장 깔끔한 UX)

단, 운영상 항상 4개 고정이 필요하면 “비활성”으로 전환 가능

FR-2: 테마 토글(다크/화이트)

우측 상단 토글로 Dark ↔ Light 전환

기본 테마: Dark

테마 저장: localStorage에 저장하여 재방문 시 복원

key 예시: theme = "dark" | "light"

로드 시 우선순위:

localStorage 값

없으면 기본 dark

FR-3: 문의하기 버튼 + 모달 폼

“문의하기” 버튼 클릭 시 모달 오픈

모달 구성:

Title: “문의하기”

Form fields:

이름(name) - 필수

이메일(email) - 필수, 이메일 형식 검증

Actions:

Primary: “보내기”

Secondary: “취소”

모달 닫기:

X 버튼

취소 버튼

바깥 영역 클릭

ESC 키

전송 처리(‘don’t care’라서 기본값 제안)

가장 빠르고 실용적인 기본안(추천):

제출 시 mailto:로 사용자 기본 메일 앱 실행

제목/본문에 이름/이메일 포함

서버/DB 없이 즉시 동작

대안(확장 가능):

API 엔드포인트로 POST → DB/노션/구글시트/슬랙 알림

5) 상태/에러/검증 요구사항 (UX States)
5.1 링크 버튼 상태

Default: enabled

Hover/Pressed 스타일

Disabled/Hidden 정책은 위 FR-1 기본값(숨김)

5.2 문의 폼 검증

이름: 빈 값이면 에러 “이름을 입력해주세요.”

이메일: 정규식 기반 형식 검사, 실패 시 “이메일 형식을 확인해주세요.”

제출 버튼:

유효하지 않으면 비활성(또는 제출 시 에러 표시)

5.3 제출 상태

(mailto 기본안) 제출 클릭 시:

검증 통과 → mailto 실행 → 모달 닫기

실패 케이스(드묾): “메일 앱을 열 수 없습니다.” 토스트

6) 데이터/설정(간단 Config)

단일페이지이므로 아래 형태의 config로 충분

{
  "brandName": "YourBrand",
  "title": "My Links",
  "subtitle": "Connect with me",
  "links": {
    "kakaoOpenChat": "https://open.kakao.com/...",
    "youtube": "https://youtube.com/...",
    "threads": "https://www.threads.net/...",
    "linkedin": "https://www.linkedin.com/in/..."
  },
  "contact": {
    "mode": "mailto",
    "mailto": "you@example.com"
  },
  "theme": {
    "default": "dark",
    "persist": true,
    "storageKey": "theme"
  }
}
